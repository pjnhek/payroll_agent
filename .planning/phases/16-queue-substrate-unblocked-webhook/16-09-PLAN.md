<objective>
Prove the phase's headline claim, and prove it can fail.

**ROADMAP criterion #2:** *Clicking "Retrigger" on a stuck run enqueues a durable `jobs` row; killing
the worker process mid-run and draining again (a second worker, or a manual drain) completes the
retrigger without the operator re-clicking anything.*

This is the proof the whole milestone exists for, and it is the one most likely to be written
vacuously. The named vacuous twin: **a test that never actually leases the job before "killing" the
worker.** The reclaim path then never fires, the first drain simply does the work, and the test passes
on dumb luck while the reclaim SQL could be entirely broken. This repo has already shipped one
concurrency proof that passed while proving nothing. Do not ship a second.

Purpose: ROADMAP criterion #2, demonstrated able to fail.
Output: Proof 2, and the migrated live-DB operator-recovery test.
</objective>

Six steps, in this exact order. **Every step has an assertion; a step without one is where the vacuity
gets in.**

1. Seed a run in `ERROR`. Note its `reply_epoch`. POST `/runs/{id}/retrigger` (or call `retrigger()`
   directly — either is fine here, because the thing under test is the QUEUE, not an HTTP race, so
   there is no TestClient-serialization hazard).

2. **Assert the durable enqueue.** A `jobs` row exists with `state='pending'`, `kind='run_pipeline'`,
   `attempts == 0`, and `dedup_key == f"run_pipeline:{run_id}:{epoch_after_clear}"`. Assert the
   orchestrator spy has NOT been called (workers are off — `WORKER_COUNT=0` — so nothing has run).

3. **Simulate a worker that claims the job and then dies mid-run.** Call `repo.claim_job()` DIRECTLY —
   **NOT `drain_once()`**. This is the whole trick: `drain_once()` would run the handler to
   completion, and you need to stop MID-LEASE. Then assert `attempts == 1`, `state == 'leased'`, the
   `lease_token` is non-null, and `leased_until` is in the FUTURE. **This assertion is what makes the
   proof non-vacuous** — it is the "the job was genuinely leased before the crash" precondition whose
   absence is the named vacuous twin. Optionally advance the run to `EXTRACTING` here to model a worker
   that got partway through before dying; that is what forces the D-01 rewind to be exercised rather
   than trivially skipped.

4. **Expire the lease WITHOUT sleeping:** `UPDATE jobs SET leased_until = now() - interval '1 second'
   WHERE id = %s`. This exercises the exact `leased_until < now()` predicate the claim SQL depends on,
   and it keeps the proof deterministic. `LEASE_SECONDS` is 900 — sleeping is not an option, and
   shortening the lease for the test would be testing a different system.

5. **Call `drain_once()`** — this is "a second worker, or a manual drain", the criterion's own wording,
   and it is the exact function the Phase-17 pump and the worker threads both call.

6. **Assert all four outcomes, separately:**
   - the job was RECLAIMED: `attempts == 2` (not 1 — a job that was never leased would show 1 here, and
     that discrepancy is your vacuity detector);
   - the D-01 rewind FIRED and the run genuinely re-ran: the orchestrator spy WAS called, and the run's
     final status is what a real re-run produces (not left at `EXTRACTING`);
   - `reply_epoch` is UNCHANGED from step 3 (**D-02** — the operator's retrigger bumped it once; the
     automatic reclaim must bump it zero times);
   - the job's final `state == 'done'`.

Write both falsifying mutations into the test's docstring and EXECUTE each:
(a) strip `OR (state='leased' AND leased_until < now())` from `claim_job` -> the job is never
    reclaimed, step 5's drain claims nothing, and the test reds;
(b) strip the `attempts > 1` rewind preamble from `handle_run_pipeline` -> the run stays at
    `EXTRACTING`, the `RECEIVED -> EXTRACTING` CAS fails, the job is marked `done`, and the run is
    STRANDED FOREVER — the test reds on "the run actually re-ran".
