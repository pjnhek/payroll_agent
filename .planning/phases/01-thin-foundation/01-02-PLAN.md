app/db/bootstrap.py: a runnable script (`if __name__ == "__main__"`) and importable function `bootstrap(reset: bool = False)`.

The bootstrap function:
1. Reads DATABASE_URL from config (get_settings().database_url). Never from a CLI positional arg.
2. Opens a single psycopg connection (not the pool — bootstrap is a one-shot admin operation) with `prepare_threshold=None`.
3. If `reset=True` (opt-in only): DROP TABLE each table in reverse dependency order (paystub_line_items → eval_results → payroll_runs → employees → email_messages → businesses, or use CASCADE). Print a clear warning "RESET: dropping all tables — this is destructive" before executing. The DROP path is inside an explicit `if reset:` block, completely separate from the default path.
4. Execute the contents of `app/db/schema.sql` using `conn.execute(sql_text)` in a transaction (all 6 tables created atomically — either all succeed or none do).
5. Print "Bootstrap complete. Tables applied." on success.

CLI entry: `python -m app.db.bootstrap` (no args = non-destructive apply). `python -m app.db.bootstrap --reset` = opt-in destructive reset. Parse `--reset` from sys.argv. No argparse dependency needed; just `"--reset" in sys.argv`.

Security note: the --reset flag must never be the default. The function signature `def bootstrap(reset: bool = False)` encodes this. The CLI check is `reset = "--reset" in sys.argv`. The destructive path prints a warning including the target host extracted from DATABASE_URL (strip the password first before printing) so the developer can visually confirm they're targeting the right DB.

tests/test_status_drift.py: a pytest test that is the CI gate for D-03 (Python RunStatus is source of truth; SQL mirrors it — drift fails CI). The test:
1. Reads app/db/schema.sql as a string.
2. Uses a regex to extract the values from the status CHECK constraint: `re.search(r"CHECK\s*\(\s*status\s+IN\s*\((.*?)\)\s*\)", sql, re.DOTALL | re.IGNORECASE)`. Extract the CSV inside the IN(...) parens.
3. Parse each value by splitting on commas, stripping whitespace, and stripping single quotes.
4. Assert the extracted set equals `{m.value for m in RunStatus}`.

This is a pure static file test — no DB connection needed. It runs in CI on every push with no env var requirements.

Use `grep -v '^--'` mental model: the regex must skip SQL comments (lines starting with --). Use `re.sub(r'--[^\n]*', '', sql)` to strip line comments before running the CHECK regex so a commented-out old value doesn't poison the check.
