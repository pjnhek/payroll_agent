IMPLEMENTATION LOGIC: The seed() function:
1. Constructs Employee model objects and validates them via Pydantic (construction raises ValidationError if any FOUND-06 field is wrong — fails at seed time per D-10).
2. If dry_run=True, returns the validated objects and exits without DB access.
3. Opens a connection via get_connection() from app.db.supabase.
4. Upserts businesses via INSERT ... ON CONFLICT (contact_email) DO UPDATE SET name=EXCLUDED.name, pay_period=EXCLUDED.pay_period, updated_at=now() (per D-11 natural key).
5. Upserts employees via INSERT ... ON CONFLICT (business_id, full_name) DO UPDATE SET known_aliases=EXCLUDED.known_aliases, pay_type=EXCLUDED.pay_type, hourly_rate=EXCLUDED.hourly_rate, annual_salary=EXCLUDED.annual_salary, ... (all mutable fields). The natural key for the unique constraint must be added to schema.sql: UNIQUE (business_id, full_name) on employees.
6. Persists jsonb fields (if any future employee fields are jsonb) via model_dump(mode="json") per D-06.
7. NEVER inserts into payroll_runs or email_messages (D-11 containment) — this is a hard constraint; add an assertion or comment.
8. Prints summary: "Seeded 3 businesses, 6 employees." on success.

Note: schema.sql must have UNIQUE(business_id, full_name) on employees for the ON CONFLICT clause to work. If Plan 02's schema.sql did not include this constraint, add it now as an ALTER TABLE ADD CONSTRAINT IF NOT EXISTS in the bootstrap script, or add it to schema.sql (the schema is the source of truth and bootstrap re-applies it).
