**Reshaped the shared Pydantic contracts to a deterministic, confidence-free interface — NameMatchResult is now source/resolved, Decision is the 4 kept fields + per-name resolutions, PaystubLineItem dropped match_confidence, and the layer-2 NameReconciliationResponse wrapper is deleted.**

## Accomplishments
- `NameMatchResult` (roster.py): dropped `confidence` (Decimal) and `match_type` (5-value Literal); added `source: Literal["exact","alias","none"]` and an explicit `resolved: bool`. Kept `submitted_name`, `matched_employee_id: UUID | None`, `reason`, and `extra="forbid"`. Docstring rewritten as the pure-code deterministic-resolution result (D-21-01/04).
- `Decision` (contracts.py): removed `model_action`, `gate_triggered`, `confidence`, `reasons`; kept `final_action`, `gate_reasons`, `unresolved_names`, `missing_fields`; added `resolutions: list[NameMatchResult]` folded into the decision JSONB (D-21-04/06). `final_action` is now the sole branch source with no model action to diverge from.
- `PaystubLineItem` (contracts.py): removed `match_confidence` and its comment block — provenance is `employee_id + submitted_name`.
- Deleted `app/models/reconcile.py` (`NameReconciliationResponse` — the dead layer-2 LLM wrapper, D-21-05).
- Rewrote `tests/test_models_contracts.py` to the new shapes: deterministic Decision tests, a `resolutions` JSON-serialization test, legacy-kwarg rejection (model_action/confidence/gate_triggered/reasons + match_type/confidence on NameMatchResult + match_confidence on PaystubLineItem), and a reconcile-module-gone test. 37 contract tests pass.

## Decisions Made
- **resolved is explicit, not derived** (D-21-04): the dashboard/eval read `resolved` directly rather than re-deriving `source != "none"`, keeping the audit record legible.
- **Decision.resolutions reuses NameMatchResult** rather than a parallel resolution-dict type — DRY, and the JSONB round-trips the same shape the resolver returns. This adds a `contracts.py -> roster.py` import (no cycle: roster.py does not import contracts).
- **Grep-clean docstrings**: the per-task verify and the CONTEXT phase-level success criterion both assert `grep ... app/models/` returns nothing. Docstrings/comments were worded to describe the absence of confidence/match_type WITHOUT using the literal tokens, so both checks pass literally rather than only in spirit.
