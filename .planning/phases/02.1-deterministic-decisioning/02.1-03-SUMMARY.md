- **Tests (both tasks):** `test_orchestrator_states.py` scripts carry only extract (+ draft) calls, the `model_action` test was rewritten to assert unresolved→clarify, and the WR-01 guard is driven at the unit level. `test_persistence.py` Section 2 uses the source/resolved + deterministic-Decision shapes and asserts the reconciliation JSONB is confidence-free. `test_threading.py` deletes the dead layer-2 reconcile/decide mock JSON, the `_stub_matches`/`_stub_decision_process` helpers use the new shapes, and the deterministic gate runs on the fixture inputs. `test_llm_client.py` routes via the draft tier and asserts the decision tier now raises. `test_status_drift.py` is unchanged and green (the schema change touches no CHECK value set).

## Decisions Made

- **bootstrap drops name_matches on the DEFAULT apply path, not only --reset** — the live DBs carry the table from Phase 2, and `CREATE TABLE IF NOT EXISTS` is a no-op on it; the only way to retire it on a running database is an explicit `DROP TABLE IF EXISTS ... CASCADE` before re-applying schema.sql. `IF EXISTS` makes it a no-op once gone.
- **The WR-01 process-run roster-integrity test moved to the unit level.** The old test tricked the gate by having the layer-2 LLM route a name to a ghost employee id; that path no longer exists (the resolver only ever resolves to a roster employee, D-21-05). The defensive `_compute_line_items` guard still matters for a stale persisted reconciliation / wrong-business roster, so it is exercised by calling `_compute_line_items` directly with a resolved match pointing outside the roster.
- **A test asserts the decision tier now raises** (`tier="decision"` → `ValueError: unknown tier`), pinning that the mid model was genuinely retired (not merely unused).
