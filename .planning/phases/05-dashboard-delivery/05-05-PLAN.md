## Phase Goal

**As an** operator, **I want to** approve payroll runs through a race-safe gate, **so that** duplicate approvals are structurally impossible and a failed delivery never silently strands a run.

<objective>
The delivery path + hardened operator routes slice. This is Beat 1: the operator clicks "Approve & Send", claim_status atomically claims the run, _deliver composes the confirmation email + generates per-employee PDFs + sends, and the run advances APPROVED → SENT → RECONCILED. A delivery failure routes to ERROR (never stuck). The retrigger route recovers from both ERROR and APPROVED states.

After this plan: a complete end-to-end approval flow works — from the operator's click through confirmation email delivery. The dashboard UI (Plan 06) will surface this in HTML; here it wires as JSON/redirect routes.

Purpose: Delivery path correctness and idempotency (HITL-02/03, CLAR-04, INGEST-05, D-13b/c, FOUND-04).
Output: _deliver in orchestrator.py, hardened approve/reject/retrigger in main.py, intent-row in gateway.py, 2 new repo helpers.
</objective>
