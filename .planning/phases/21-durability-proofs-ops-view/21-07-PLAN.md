<objective>
Wire D-13's predicate into the second of its two firing places: a cron-checkable health endpoint,
added to `pump.yml` as a failing `curl -f` step positioned so it can never suppress the drain
(D-14, D-15).

Purpose: the swallowing bug is by definition *the failure nobody noticed*. A banner on a page you
would only visit if already suspicious does not change that. This is exactly the pattern Phase 17
insisted on carrying forward for `/health/schema` — which STATE.md calls the only monitor that
catches a manual Supabase edit bypassing deploy-migrate.

Purpose (D-15): in `pump.yml` the alarm step is last and the drain executes regardless of its
result. An alarm placed ahead of the drain would turn "something went wrong" into "and now nothing
recovers" — and this repo has already been bitten once by a `pump.yml` `if:`-guard gap that code
review caught. **Recovery first, reporting second**, and the drain must be *verified* to run while
the alarm is firing, because an ordering bug is invisible when the alarm is quiet.

Output: a new health route, a workflow step, automated ordering pins, and a human verification of
the one property no unit test can establish.

**Correction applied from cross-AI review (Codex, MEDIUM ×2).** Two dependencies made explicit:

1. **This endpoint inherits plan 21-02's predicate wholesale, so it inherits its correctness.** The
   review found that predicate's original `>=` correlation had a false negative — an error a later,
   unrelated settlement would silently vouch for. A green `/health/queue` built on that would be a
   *misleading* green: the alarm reporting all-clear on precisely the swallowed error it exists to
   surface. This plan is therefore **gated on 21-02's corrected equality predicate and its
   late-no-op-job regression test**. Before writing the route, confirm in `21-02-SUMMARY.md` that
   the equality correlation shipped and that the late-no-op-job test is present and passing. If it
   is not, stop and report — do not wire a cron alarm to an unproven predicate.
2. **The live baseline must be dispositioned before the cron alarm goes live.** D-16 rules out a
   mute by design, so any pre-existing unaccounted error rows will red the scheduled pump every 30
   minutes from deploy. The checkpoint below now requires the operator's disposition of those rows
   to be **recorded** — retriggered, terminally settled, or intentionally retained — as a condition
   of approval, not as a follow-up.
</objective>
