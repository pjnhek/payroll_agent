<objective>
Wire D-02's completeness gate into CI and prove it non-vacuous against the live repository.

Purpose: PROOF-05's premise as written in REQUIREMENTS is stale. `concurrency-proof.yml` no longer
hard-codes the durability proofs by filename — Phase 16's D-14 already added a marker-selected
`-m queueproof` step that collects any marked test anywhere under `tests/` with zero workflow
edits. What remains is the residual gap the workflow names verbatim in its own comment: *it cannot
detect a typo'd marker on one newly-added test while the other queueproof tests still pass — the
log still says "N passed" and the new proof silently never ran. Nothing in CI closes that.* D-02
closes exactly that, at the selection layer, which is where the gap lives.

This plan runs in wave 4 by necessity: the gate asserts all four PROOF ids exist, so wiring it
before plans 21-03/04/05/08 had tagged them would have redded CI for the rest of the phase.

Output: a third CI step, the live-repository assertion, workflow pins, and an executed proof that
the gate reds on a real typo.
</objective>
