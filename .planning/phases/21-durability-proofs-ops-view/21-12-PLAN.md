<objective>
Restore `.github/workflows/ci.yml` to green. That workflow has been red since Phase 20 and its
failure is invisible to local development, because a developer's `.env` supplies the very variable
CI deliberately withholds.

Two independent root causes, 17 failing tests:

**H1 — 16 tests, `pydantic_core.ValidationError: 1 validation error for Settings`.**
`app/config.py:27` declares `database_url: str` with no default ("fails fast if unset"), and
`app/config.py:151` loads `env_file=".env"` relative to cwd. CI checks out a bare tree with no
`.env` and sets no `DATABASE_URL` (`ci.yml:67-68`, deliberate — the conftest two-factor guards are
supposed to skip DB tests hermetically). Any test that reaches `get_settings()` therefore raises.
`tests/conftest.py::mock_llm` already stubs `DATABASE_URL` for precisely this reason and says so in
its docstring ("test environments that lack a .env file (worktrees, bare CI, etc.)") — the 16
failures are simply the tests that do not take that fixture.

**H2 — 1 test, `FileNotFoundError: 'pdftotext'`.**
`tests/test_pdf.py:119` shells out to `pdftotext` (poppler-utils) to extract real text from a
generated paystub. The GitHub runner does not ship it. This is a CI provisioning gap, not a test
defect.

Output: one suite-wide settings stub that is inert whenever a real `DATABASE_URL` exists, and a
poppler-utils install step in the hermetic job.
</objective>

<critical_context>
**This plan does not touch the 16 failing test files.** The correct fix is one fixture in
`tests/conftest.py`, not 16 edits. If you find yourself editing `tests/test_gateway.py`,

**H2 has one forbidden resolution.** Do NOT make the PDF test skip when `pdftotext` is absent.

today. Install the dependency.

**Do not weaken the two-factor live-DB guard.** `tests/conftest.py:114-128` requires BOTH
`DATABASE_URL` and `ALLOW_DB_RESET=1`. The stub you add must not become a way for live-DB tests to
start running (or half-running) in the hermetic job. Truth #3 is the guard on this and must be
demonstrated, not asserted.
</critical_context>
