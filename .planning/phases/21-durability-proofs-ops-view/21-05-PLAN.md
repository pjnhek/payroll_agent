<objective>
Build PROOF-03, the one proof with no incumbent coverage: crashing between Resend accepting a send
and the local `sent` commit results in zero second emails, with the persisted `message_id`
asserted byte-identical across both attempts (D-03).

Purpose: ROADMAP criterion 3 and REQUIREMENTS both warn this proof is **vacuous if it passes
against a fake gateway while SEND-01 is unfixed**. SEND-01/02/03 already shipped in Phase 20, so a
naive "did it send successfully" test cannot distinguish "the fix works" from "the fix was never
load-bearing here." D-03 states the teeth exactly: **the persisted `message_id` is byte-identical
across both attempts, exactly one provider call was made, and it carried the same
`Idempotency-Key`.** Those three assertions are the proof; the crash mechanism is not.

Purpose (mechanism): D-03 mandates an injected-seam failure against a real Postgres, not a hard
connection kill. A hard kill proves the same invariant while being nondeterministic under a shared
CI Postgres, and a flaky proof is a proof that eventually gets quarantined.

Output: a new live-Postgres proof carrying the PROOF-03 id, plus publishable mutation evidence.
</objective>
