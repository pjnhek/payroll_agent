1. **`_DEMO_FIXTURE_DEFAULT_KEY` omitted from the demo constants move list (Codex Round 2 HIGH,
   CONFIRMED this session):** `_DEMO_FIXTURE_DEFAULT_KEY = "coastal_exact"` is defined at
   `app/main.py:149` (immediately after the `_DEMO_FIXTURES` dict literal) and used as
   `demo_send_test`'s `Form` default value at `app/main.py:1754`
   (`fixture_key: str = Form(default=_DEMO_FIXTURE_DEFAULT_KEY)`). The prior revision's demo
   constants move list named `_DEMO_FIXTURES`/`DEMO_OPERATOR_EMAIL`/`_SEED_CONTACTS`/
   `_SEED_BUSINESS_IDS` but omitted this one -- once `demo_send_test` moves to `demo.py` without
   also moving this constant, `demo.py` raises `NameError` the moment Python evaluates the function
   signature's default value at IMPORT time (not even at call time), breaking the entire app's
   startup. Fixed here: `_DEMO_FIXTURE_DEFAULT_KEY` -> `DEMO_FIXTURE_DEFAULT_KEY` (promoted public,
   same reasoning as the other three demo constants -- it's referenced as a signature default that
   must be resolvable wherever `demo_send_test` lives) moves to `demo.py` in the SAME promotion pass.
2. **`test_demo_landing.py`'s `raising=False` dead-attribute patches (Codex Round 2 HIGH, CONFIRMED
   this session -- and independently flagged by this round's planning context as a security-relevant
   threat):** `tests/test_demo_landing.py` patches `app.main._run_pipeline` with `raising=False` at
   THREE sites (confirmed this session at lines 812, 877, 934, inside
   `test_compose_...`/similar tests). `raising=False` means the patch SUCCEEDS even if
   `app.main._run_pipeline` no longer exists (which it won't, post-split -- `demo_compose` and
   `demo_send_test` move to `demo.py` and call `pipeline_glue.run_pipeline_bg`). Left unfixed, these
   three patches would silently become no-ops: the test would set an attribute on a module that no
   route consults, then the ACTUAL route would call the REAL `pipeline_glue.run_pipeline_bg`,
   invoking the real pipeline -- and since this repo's `.env` carries LIVE LLM keys (per
   STATE.md's "Execute-phase integration hazards" finding), this could silently trigger a real
   LLM call inside what is meant to be a hermetic test. Fixed here: all three sites retarget to
   `monkeypatch.setattr(pipeline_glue_mod, "run_pipeline_bg", ..., raising=True)` (module import
   `import app.routes.pipeline_glue as pipeline_glue_mod`) -- `raising=True` (the default) is used
   deliberately here so that if a FUTURE refactor renames `run_pipeline_bg` again, this patch fails
   LOUDLY instead of silently no-op'ing a second time. See Task 2 and this plan's `<threat_model>`.
3. **`FileResponse` unused-import cleanup in `runs.py` (Codex Round 2 LOW, confirmed this session):**
   verified via `rg -n "FileResponse" app/main.py` that `FileResponse` is used exactly once, at
   line 1575, inside `eval_chart` (moving to `dashboard.py`). `paystub_pdf` (moving to `runs.py`)
   uses `StreamingResponse`, not `FileResponse` (confirmed by reading its full body this session).
   `runs.py`'s import list in this plan does NOT include `FileResponse`.
