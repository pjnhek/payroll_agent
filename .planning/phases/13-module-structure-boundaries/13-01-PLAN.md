**`_conn_ctx`/`get_connection` fix (Codex-confirmed gap):** `app/db/repo/_shared.py`'s `_conn_ctx`
must NOT do a module-level `from app.db.supabase import get_connection`. Instead, resolve
`get_connection` through the package at CALL TIME (a function-body lazy import, safe because the
package is fully initialized by the time any request-handling code calls `_conn_ctx` -- the
forbidden case is a MODULE-LEVEL import of the package from within one of its own submodules,
which is what would create a circular-import-at-init-time failure):
```
def _conn_ctx(conn):
    if conn is not None:
        yield conn, False
    else:
        import app.db.repo as _repo_pkg
        with _repo_pkg.get_connection() as owned:
            yield owned, True
```
This makes `monkeypatch.setattr(repo, "get_connection", ...)` (the facade -- the exact seam
`tests/conftest.py`'s `fake_repo` fixture already patches) intercept `_conn_ctx`'s fallback path
identically to pre-split behavior, because `_repo_pkg` IS the same `app.db.repo` module object
`monkeypatch` patches. The facade's own `__init__.py` re-exports `get_connection` directly from
`app.db.supabase` (see below) so this call-time lookup resolves to the SAME patched attribute
whichever way a test approaches it.

**Same-aggregate internal calls (record_run_error->set_status, update_email_message_sent-
>update_email_message_state):** keep these as plain same-module bare-name calls -- do NOT route
them through a package-object lookup. Same-module bare-name resolution is UNCHANGED by the split
(Python resolves an unqualified name via the enclosing module's own global namespace at call time,
regardless of whether that module is now one-of-six instead of one-of-one) as long as both
functions in each pair stay in the SAME new file, which D-02 already guarantees (`record_run_error`
