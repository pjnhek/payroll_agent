**`_conn_ctx`/`get_connection` fix:** `app/db/repo/_shared.py`'s `_conn_ctx` must NOT do a
module-level `from app.db.supabase import get_connection`. Instead, resolve `get_connection`
through the package at CALL TIME (a function-body lazy import, safe because the package is fully
initialized by the time any request-handling code calls `_conn_ctx` -- the forbidden case is a
MODULE-LEVEL import of the package from within one of its own submodules, which is what would
create a circular-import-at-init-time failure):
```
def _conn_ctx(conn):
    if conn is not None:
        yield conn, False
    else:
        import app.db.repo as _repo_pkg
        with _repo_pkg.get_connection() as owned:
            yield owned, True
```
This makes `monkeypatch.setattr(repo, "get_connection", ...)` (the facade -- the exact seam
`tests/conftest.py`'s `fake_repo` fixture already patches) intercept `_conn_ctx`'s fallback path
identically to pre-split behavior, because `_repo_pkg` IS the same `app.db.repo` module object
`monkeypatch` patches, and `_conn_ctx` itself lives in `_shared.py` -- a DIFFERENT module than the
facade, so it genuinely needs this call-time indirection (unlike `record_run_error`/`set_status`,
which are same-module and need no indirection at all).
