<objective>
Close POLISH-01's two remaining sanctioned behavior fixes, test-first: WR-05 (path containment
on eval fixture reads) and INFO-02 (ValidationError content echoed back to the LLM provider),
per D-12.

WR-05 carries one extra, review-mandated step. The obvious test setup — `monkeypatch.chdir` +
`GET /eval` — CANNOT work: `app/routes/templating.py` builds one shared
`Jinja2Templates(directory="app/templates")` with a RELATIVE searchpath, so changing the working
directory makes `eval.html` unresolvable (TemplateNotFound). And `eval_view` builds
`Path("eval/summary.json")` and `Path("eval/fixtures")` as function-local variables, so there is
nothing to monkeypatch. The fix is to hoist those two paths to module-level constants — the same
shape `eval/run_eval.py` already uses for FIXTURE_DIR / SUMMARY_PATH, and the same seam
`tests/test_eval_wiring.py` already monkeypatches — and let the test redirect the constants
instead of the process.

Purpose: WR-01's proof lives in plan 15-01; this plan is the other half of POLISH-01's code work.
These plus 15-01 are the ONLY sanctioned behavior changes in Phase 15 — everything else in the
phase is comment/docstring/rename-only.
Output: two small production fixes with their regression tests, committed test-first, with the
suite, ruff, and strict mypy green at every commit.
</objective>
