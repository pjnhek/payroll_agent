  CRITICAL SECURITY RULES (per D-17 and planning_context threat model):
  - The push job (check job) MUST NOT reference any secrets — it is fully hermetic
  - DATABASE_URL must be set to a placeholder literal "placeholder" in the check job env — otherwise app.config fails fast on startup when run_eval.py calls get_settings() in _write_summary_json; however since --check mode exits before _write_summary_json, DATABASE_URL may not even be needed if run_eval.py defers get_settings() correctly; provide it as "placeholder" regardless to be safe
  - ALLOW_LIVE_LLM is intentionally NOT set in the push job — absence means allow_live_llm=False (the default in config)
  - Only the workflow_dispatch record job references secrets (EXTRACTION_API_KEY), and only when live_record input is true

  WORKFLOW STRUCTURE:

  name: eval

  on:
    push:
      branches: ["master"]
    workflow_dispatch:
      inputs:
        live_record:
          description: "Re-record extraction cache (requires EXTRACTION_API_KEY secret)"
          type: boolean
          default: false
          required: false

  jobs:
    check:
      name: "Eval regression gate (hermetic)"
      runs-on: ubuntu-latest
      # Push and all workflow_dispatch runs get the hermetic check job
      steps:
        - name: Checkout
          uses: actions/checkout@v4
        - name: Set up uv + Python 3.12
          uses: astral-sh/setup-uv@v5
          with:
            python-version: "3.12"
        - name: Install deps (all groups)
          run: uv sync
        - name: Run eval regression gate
          run: uv run python eval/run_eval.py --check
          env:
            DATABASE_URL: "placeholder"
            # NOTE: ALLOW_LIVE_LLM intentionally NOT set — eval gate stays hermetic on push.
            # Green CI = deterministic scoring didn't regress against committed summary.json.
            # Green CI does NOT mean extraction is good (no live LLM was called).

    record:
      name: "Live re-record (workflow_dispatch only)"
      if: github.event_name == 'workflow_dispatch' && inputs.live_record
      runs-on: ubuntu-latest
      steps:
        - name: Checkout
          uses: actions/checkout@v4
        - name: Set up uv + Python 3.12
          uses: astral-sh/setup-uv@v5
          with:
            python-version: "3.12"
        - name: Install deps (all groups)
          run: uv sync
        - name: Re-record extraction cache
          run: uv run python eval/run_eval.py --record
          env:
            DATABASE_URL: "placeholder"
            ALLOW_LIVE_LLM: "true"
            EXTRACTION_API_KEY: ${{ secrets.EXTRACTION_API_KEY }}

  VERIFY the push job has zero secret references: grep the file for `secrets.` — must return only the record job block, never the check job block.

  DO NOT add a keep-alive cron job here — that is Phase 6 (OPS-03). This file owns ONLY the eval workflow.
