  3. Run the deterministic stages on the LABELED expected extraction (PATH A — isolated, per D-07; reconcile/validate/decide are scored on truth, NOT on cache):

  4. EXTRACTION SCORING (D-06 precision+recall+F1 over the CACHED real extraction vs labeled expected — this is the ONLY metric that touches the cache):
     Align employees by normalized submitted_name using MULTISET semantics so a duplicated extracted employee does NOT silently collapse (a duplicate pay line is a dangerous invention, D-06).
     from collections import Counter
     actual_counts  = Counter(_normalize(e.submitted_name) for e in cached_extracted.employees)
     expected_counts = Counter(_normalize(e["submitted_name"]) for e in raw["expected"]["extracted"]["employees"])
     true_positives = sum((actual_counts & expected_counts).values())   # multiset intersection
     false_positives = sum((actual_counts - expected_counts).values())  # extras + duplicates = hallucinated/invented lines
     false_negatives = sum((expected_counts - actual_counts).values())  # dropped employees
     precision = true_positives/(true_positives+false_positives) if (true_positives+false_positives) > 0 else 1.0
     recall = true_positives/(true_positives+false_negatives) if (true_positives+false_negatives) > 0 else 1.0
     f1 = (2*precision*recall/(precision+recall)) if (precision+recall) > 0 else 0.0
     
     FIELD ACCURACY (first-class, EVAL-04 — "extraction field accuracy", reported ALONGSIDE F1, not hidden behind it):

     Count field_correct/field_total across those matched employees.
     extraction_scores = {"precision": precision, "recall": recall, "f1": f1, "field_accuracy": field_correct/field_total if field_total > 0 else 1.0, "field_correct": field_correct, "field_total": field_total, "true_positives": true_positives, "false_positives": false_positives, "false_negatives": false_negatives}

  AGGREGATION FUNCTION: _aggregate(fixture_results: list[dict]) -> dict
  Computes per-category metrics + confusion matrix.

  EXTRACTION per-category:
  Group fixture_results by fixture_category. For each category: mean F1, mean field_accuracy.
  Also compute overall extraction F1 (macro-average across fixtures).

  RECONCILIATION per-NAME-category:
  Flatten all reconciliation_results entries across all fixtures.
  Group by name_category. For each: count correct / total (the k/n fraction, D-12).
  Store as {"category": c, "correct": k, "total": n, "accuracy": k/n}.

  DECISION confusion matrix (D-11, D-12):
  Build a 2×2 matrix over all fixtures (classify each fixture by actual decision.final_action vs expected final_action):
    actual_process & expected_process    = true_process  (correct process)
    actual_process & expected_clarify     = false_process (THE HEADLINE COUNT — leaked through and pays the wrong person)
    actual_clarify & expected_process     = false_clarify (annoying, not dangerous)
    actual_clarify & expected_clarify      = true_clarify (correct clarify)

  The HEADLINE is the false_process COUNT (an integer, unambiguous at any n). Report TWO rates alongside it, clearly distinguished — do NOT let the precision-style rate masquerade as the risk number (Codex HIGH finding):
    - false_process_rate (PRIMARY / risk framing — "of the cases that SHOULD have clarified, how many leaked through and processed"):
        expected_clarify_total = false_process + true_clarify
        false_process_rate = false_process / expected_clarify_total if expected_clarify_total > 0 else 0.0
    - false_process_precision_rate (SECONDARY, labeled — "of the cases we DID process, how many were wrong"):
        actual_process_total = false_process + true_process
        false_process_precision_rate = false_process / actual_process_total if actual_process_total > 0 else 0.0

  Per-fixture-category decision: for each category, report as "k/n correct" (D-12 — use fractions, not percentages at small n).

  Return aggregated = {
    "extraction_overall_f1": float,
    "extraction_overall_field_accuracy": float (macro-mean of per-fixture field_accuracy — first-class, EVAL-04),

    "per_category_reconciliation": [{category, correct, total, accuracy}, ...],
    "confusion_matrix": {"true_process": int, "false_process": int, "false_clarify": int, "true_clarify": int, "false_process_rate": float, "false_process_precision_rate": float},
    "per_category_decision": {cat: {"correct": k, "total": n, "fraction": "k/n"}, ...},
    "rigor_gate_struct_accuracy": float (fraction of fixtures where gate_struct_ok=True)
  }
