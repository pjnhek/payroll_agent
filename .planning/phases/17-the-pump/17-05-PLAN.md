<objective>
Build the phase's single most important test (criterion #2, PUMP-01) — the anti-vacuous-proof anchor. It
constructs exactly the state the roadmap names: a job scheduled for later (future `available_at`), on an
instance with NO live worker threads, and proves the pump — hit via the HTTP endpoint cron actually calls —
executes it. This project's history is dominated by vacuous proofs (a "concurrency proof" that passed while
proving nothing); this test is deliberately engineered to be non-vacuous and is demonstrated able to fail.

The suite already pins `WORKER_COUNT=0` (tests/conftest.py), so every TestClient runs cold-started by
construction — the state criterion #2 wants is the default, not elaborate staging. The design (RESEARCH.md
§ "Designing the anti-vacuous-proof anchor") makes the four vacuity traps impossible:
(a) not requesting `live_worker` (no worker races ahead), (b) asserting the job is genuinely not-claimable
while future-dated (the claim query isn't trivially satisfied), (c) asserting claimed==1 + the specific
outcome + the row reaching state='done' by its own id (not merely a 200), and (d) — review finding #3 —
STUBBING `pipeline_glue.run_pipeline_now` before the endpoint drains it, since the seeded job is kind
`run_pipeline` and an unstubbed drain would invoke the REAL orchestrator and paid LLM providers
(pipeline.py:159) AND could let `done==1` pass on a run that ended in ERROR (the pipeline catches stage
failures and returns normally, pipeline.py:74). The stub records `orchestrator_calls` and advances the run
to a terminal status, and the test asserts `orchestrator_calls == [run_id]` (the correct handler ran exactly
once) — a handler-side observable, not just a row state. This reuses the exact stub-and-observe pattern the
sibling proof at test_queue_durability.py:975/1018 already established.

This plan also delivers the LIVE half of count_open_jobs that 17-02's FakeConnection test could not prove: a
second queueproof assertion builds a real pending/leased/done/dead mix and checks the exact open count, plus
`queue_depth == 0` once the pump's sole job completes.

Purpose: prove the pump, not the workers, is the durable-execution guarantee — and prove the proof can fail.
Output: @pytest.mark.queueproof tests in tests/test_queue_durability.py (the stubbed pump anchor + the live
mixed-state count) + the falsifying-mutation record.
</objective>
