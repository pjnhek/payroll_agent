<objective>
Enrich the shared `drain_once()` (D-04) so it surfaces each job's terminal outcome as a small
`DrainOutcome` StrEnum (`empty | done | retried | dead | fenced`) instead of a bare `bool`. This is the
foundation the pump route (17-04) aggregates into its real per-invocation counts. The change is a
capture-don't-thread edit inside `drain_once()` only — `repo.complete_job` already returns `bool` and
`repo.fail_job` already returns `JobState | None`; both values are computed inside `drain_once()` today
and discarded. No repo signature changes.

Review correction (finding #1, overturning 17-RESEARCH Open Question #1): the rare double-failure branch —
`fail_job()` itself raising — is NOT mapped to a truthy FENCED outcome. It is an infra failure (a failed DB
write that intentionally retains the lease, drain.py:180), semantically distinct from a settled fence
(drain.py:7 — a write that landed but matched zero rows). It re-raises out of `drain_once()` so the pump
route returns 503 (D-10) and the worker loop (worker.py:203) still catches it and survives. FENCED stays
reserved for a settled fence. The 5-value DrainOutcome vocabulary (D-04) is unchanged — an infra failure is
an exception, not a sixth outcome.

The single largest mechanical cost is the ~15 existing `assert drain.drain_once() is True/False`
identity assertions across six test files: `is True` is an identity check against the `bool` singleton
and fails for ANY non-bool object regardless of its truthiness. Each is rewritten to assert the
*specific* expected DrainOutcome (a proof-strengthening, not just a repair).

Purpose: turn the queue's single drain step into something whose result the pump can count truthfully.
Output: `DrainOutcome` + `drain_once() -> DrainOutcome` (re-raising the double-failure), all identity
assertions rewritten.

Note (review low-finding, 17-01 TDD label): Task 1 carries a `<behavior>` block as its task-level TDD
contract; the outcome-specific behavioral assertions land in Task 2, the dedicated test task. The split is
intentional (Task 1 is production + its own import-level shape/mypy proof; Task 2 owns the behavioral
assertions across seven files), not an oversight.
</objective>
