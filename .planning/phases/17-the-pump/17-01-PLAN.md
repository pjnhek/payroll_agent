<objective>
Enrich the shared `drain_once()` (D-04) so it surfaces each job's terminal outcome as a small
`DrainOutcome` StrEnum (`empty | done | retried | dead | fenced`) instead of a bare `bool`. This is the
foundation the pump route (17-04) aggregates into its real per-invocation counts. The change is a
capture-don't-thread edit inside `drain_once()` only — `repo.complete_job` already returns `bool` and
`repo.fail_job` already returns `JobState | None`; both values are computed inside `drain_once()` today
and discarded. No repo signature changes.

The single largest mechanical cost is the ~15 existing `assert drain.drain_once() is True/False`
identity assertions across six test files: `is True` is an identity check against the `bool` singleton
and fails for ANY non-bool object regardless of its truthiness. Each is rewritten to assert the
*specific* expected DrainOutcome (a proof-strengthening, not just a repair).

Purpose: turn the queue's single drain step into something whose result the pump can count truthfully.
Output: `DrainOutcome` + `drain_once() -> DrainOutcome`, all identity assertions rewritten.
</objective>
