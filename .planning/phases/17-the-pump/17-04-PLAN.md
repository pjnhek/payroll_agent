<objective>
Add the authenticated pump endpoint (PUMP-01, criterion #1). `GET /internal/pump` is a thin, sync `def`
route modeled on app/routes/health.py: it authenticates with a constant-time Bearer compare
(hmac.compare_digest, D-01) that fails closed on an unset/empty PUMP_TOKEN (D-03) and returns 401 on any
bad/missing credential (D-02, deliberately not 404). It then loops the SAME `drain_once()` the worker
threads call (PUMP-01 — never a fork), bounded by a max-jobs cap AND a wall-clock cap checked between jobs
(D-05), aggregating each DrainOutcome into real counts, and returns
`{claimed, done, retried, dead, fenced, queue_depth}` (queue_depth via repo.count_open_jobs). Business
outcomes (dead-letter, backoff) return 200 + counts (D-09); only a genuine infra outage returns 503 (D-10).

Discretion calls (documented at the call site): method is GET (simplest for a curl cron; the drain is
idempotent — SKIP LOCKED makes repeat/concurrent hits safe); env var is PUMP_TOKEN / setting pump_token;
caps are `_MAX_JOBS_PER_PUMP = 20` and `_MAX_WALL_CLOCK_SECONDS = 120` (derivation: worst-case single job
is 210s per app/config.py LEASE_SECONDS; the wall-clock cap is checked BETWEEN drain_once() calls so a
pump request runs at most ~330s, which pump.yml's `--max-time 360` covers); queue_depth is a separate
cheap SELECT, not part of the claim transaction.

Purpose: the pump is the PRIMARY execution trigger — it must drain with zero live worker threads.
Output: /internal/pump route, pump_token setting, router wiring, hermetic auth/bounded/infra tests.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| cron/client → /internal/pump | Untrusted network request; the only trusted input is the Bearer secret, compared before any drain. |
| pump route → DB (claim/count) | A DB outage here must surface RED, not be swallowed into a GREEN 200. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-17-01 | Information Disclosure | _authorized() Bearer compare | medium | mitigate | `hmac.compare_digest(bytes, bytes)`, never `==` — timing-attack-resistant by construction (D-01, V2/V6). Pinned by the auth tests. |
| T-17-02 | Elevation of Privilege | unset PUMP_TOKEN | high | mitigate | Fail closed: `_authorized()` returns False immediately when the token is falsy, before the compare (D-03). Pinned by the empty-secret auth test. |
| T-17-03 | Denial of Service | drain loop | medium | mitigate | Dual cap — `_MAX_JOBS_PER_PUMP` (20) and `_MAX_WALL_CLOCK_SECONDS` (120, checked between jobs) bound each invocation so a poison/backlog flood cannot pin the request (D-05). Pinned by the bounded test. |
| T-17-04 | Information Disclosure | 401-vs-404 status | low | accept | 401 (not 404) deliberately reveals the route exists, in exchange for a loud RED on a misconfigured token (D-02). The route is not a data-disclosure surface. |
| T-17-05 | Information Disclosure | 503 body / logs | medium | mitigate | The infra branch logs only `type(exc).__name__` and returns a fixed "pump unavailable" body — never `str(exc)` (which could carry a connection string). Mirrors health.py disclosure discipline. |
| T-17-SC | Tampering | package installs | low | accept | Zero new packages this phase (RESEARCH Package Legitimacy Audit — not applicable). `hmac` is stdlib. |
