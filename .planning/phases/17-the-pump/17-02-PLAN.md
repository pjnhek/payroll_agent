<objective>
Add the point-in-time queue-depth read the pump reports as `queue_depth` (D-04, criterion #1's fifth
count). `count_open_jobs()` is a plain `SELECT count(*) FROM jobs WHERE state IN ('pending','leased')`
following jobs.py's uniform `_conn_ctx` read convention. It is deliberately backlog-scoped (total
outstanding), not "claimable right now" — the useful ops signal is outstanding depth. Two easy-to-miss
companions land in the same commit: the module docstring's "Six functions…" line becomes false (update to
seven + enumerate the new name), and the facade re-export in `app/db/repo/__init__.py`.

the existing deploy-migrate path. This plan adds only a read — NO schema-push task is needed or created.

Purpose: give the pump a truthful backlog number without reusing a composition snapshot as per-run counts.
