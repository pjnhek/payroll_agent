# Phase 7.5 Plan 02: RawFieldDrop + detect_field_regression + validate(raw_field_drops=) + decide Rule 2b + compose_email field-regression copy

Pure-function MONEY-03 judgment layer: `RawFieldDrop` model, `detect_field_regression` public pure helper (employee-id-keyed, R3-3 fix), `validate(raw_field_drops=)` seam (N6/N8-correct, D-7.5-10 compliant), decide Rule 2b, and the field-regression email copy (N5/D-7.5-09, deterministic regardless of LLM draft path).

## What Was Built

### Task 1: RawFieldDrop + detect_field_regression + validate(raw_field_drops=) seam

**`app/models/contracts.py` — RawFieldDrop model:**
- Inserted before FieldDrop (~line 119): `class RawFieldDrop(BaseModel)` with `ConfigDict(extra="forbid")`
- Fields: `submitted_name: str`, `field: str`, `original_value: Decimal = Field(ge=0)`, `resumed_value: Decimal | None = Field(default=None, ge=0)`
- Internal counterpart to FieldDrop: submitted_name-keyed (for orchestrator→validate seam), vs FieldDrop's employee_id-keyed (public)
- No new imports needed — UUID, Decimal, Field, BaseModel, ConfigDict already imported

**`app/pipeline/validate.py` — detect_field_regression (PUBLIC pure function):**
- New imports: `from uuid import UUID` and `RawFieldDrop` added to contracts import
- `detect_field_regression(original, resumed, prior_matches, current_matches) -> list[RawFieldDrop]`
- **R3-3 fix (employee-id-keyed diff):** builds `{employee_id: ExtractedEmployee}` maps for both original and resumed using prior_matches/current_matches BEFORE diffing — "M. Chen" (prior) and "Maria Chen" (resumed), same employee_id, now land in the same diff slot
- **Honest no-op:** returns `[]` immediately when `prior_matches is None` — documented; production always threads prior_matches
- **D-09 DRY:** reuses `_HOURS_FIELDS` and `_is_paid` — the third call site of `_is_paid`
- **D-25/D-26:** `_is_paid` covers both absent and explicit-zero → drop produced for OT 2→0 and 2→absent; `resumed_value=Decimal('0')` preserved (not coerced to None)
- **D-27:** sorted by `str(employee_id)` for deterministic ordering
- **D-7.5-10 structural enforcement:** function is PUBLIC; NOT called internally by validate()

**`app/pipeline/validate.py` — validate(raw_field_drops=) seam:**
- Signature extended: `*, prior=None, prior_matches=None, resolved_drops=None, raw_field_drops=None`
- When `raw_field_drops is not None and len(raw_field_drops) > 0`:
  - Builds `name_to_id_current` from matches
  - For each RawFieldDrop: looks up `current_emp_id`, skips if unresolved
  - **N8 suppression (KEY TYPE FIX):** `(str(current_emp_id), raw_drop.field) in _resolved_drops` — UUID→str conversion for consistency with Plan 03 Step E2's `set[tuple[str, str]]`
  - Emits `ValidationIssue(issue_type="field_regression", field=f"{submitted_name}.{field}", message=...)`
- `prior=` kwarg: kept for signature compatibility (Plan 01); not used in body; documented with comment
- `any_hours` gate: unchanged, runs on BACKFILLED data per D-7.5-10

### Task 2: decide Rule 2b + compose_email N5 + D-7.5-09 deterministic wording

**`app/pipeline/decide.py` — Rule 2b:**
- Added immediately after Rule 2 (lines ~124-125): comment + 2 lines
- `regressions = [i.field for i in issues if i.issue_type == "field_regression"]`
- `gate_reasons += [f"field regression: {f}" for f in regressions]`
- `Decision.missing_fields` NOT widened (regressions feed gate_reasons only, D-7.5 PATTERNS §4)

**`app/pipeline/compose_email.py` — _field_regression_lines helper:**
- `_field_regression_lines(gate_reasons: list[str]) -> list[str]`
- Uses `rsplit(".", 1)` (LAST-dot split) so "M. Chen.hours_overtime" → ("M. Chen", "hours_overtime"), NOT ("M", "Chen.hours_overtime")
- D-7.5-09 wording: `"  - Reply with the {field_name} hours for {submitted_name}, or 'none' to confirm zero."`

**`app/pipeline/compose_email.py` — _template_body N5 fix:**
- Unconditional `fr_lines = _field_regression_lines(decision.gate_reasons)` block BEFORE the `if not unresolved and not missing` fallback gate
- `lines.extend(fr_lines)` — field-regression lines appear alongside unresolved/missing content, not only when both are absent
- Fallback gate updated: `if not unresolved and not missing and not fr_lines` — preserves raw gate_reasons for edge cases

**`app/pipeline/compose_email.py` — compose_clarification Finding 4 fix:**
- After obtaining LLM draft body (non-empty path), appends field-regression lines:
  ```python
  fr_lines = _field_regression_lines(decision.gate_reasons)
  if fr_lines:
      body = body.rstrip("\n") + "\n\n" + "\n".join(fr_lines)
  ```
- Guarantees D-7.5-09 wording appears on the REAL (LLM-draft) path, not only template fallback
