    NEW REPO HELPER 1 — bind_demo_business(business_name: str, operator_email: str, seed_contacts: dict[str, str], conn=None) -> bool

    Atomically binds one business to the operator email and restores all others to their seed values IN ONE TRANSACTION. This is the single-owner guarantee (CHANGE-5): find_business_by_sender does exact-match, so two businesses sharing the operator email would cause ambiguous routing — the binding MUST be atomic.

    Algorithm:
    (a) In one transaction: for each (name, seed_email) in seed_contacts.items(), execute UPDATE businesses SET contact_email = %s WHERE name = %s — this resets ALL businesses including the target to their seeds.
    (b) Then in the same transaction: UPDATE businesses SET contact_email = %s WHERE name = %s with (operator_email, business_name) — this sets ONLY the target to the operator email.
    (c) Return True if the target UPDATE affected rowcount >= 1, else False (business name not found).
    (d) Use _conn_ctx(conn) as (c, owns) and a c.transaction() if owns block. Parameterized SQL only — no f-strings in execute().

    The seed_contacts dict is passed in (not hard-coded) so the function stays testable without reading the DB for seed data. The caller (the route handler) builds the dict from a module-level constant sourced from seed.py's _BUSINESSES list.

    NEW REPO HELPER 2 — load_thread_messages(run_id: uuid.UUID, conn=None) -> list[dict]

    Returns ALL email_messages rows for a run (both inbound and outbound), ordered chronologically. Columns: direction, purpose, subject, body_text, message_id, from_addr, to_addr, created_at. Uses explicit column list (no SELECT *). dict_row row factory. Ordered by created_at ASC so the thread renders oldest-first (inbound payroll request first, then clarification, then reply, then confirmation).
