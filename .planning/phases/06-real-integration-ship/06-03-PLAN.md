    STEP 1: Create a fresh Supabase project.
    - Go to app.supabase.com → New Project.
    - Note: project ref (the slug, e.g. "abcdefghijklm"), region (e.g. "us-east-1"), and database password (the one you set at creation — save it; you cannot recover it).

    STEP 2: Obtain TWO connection strings from the Supabase dashboard.
    - Supabase Dashboard → Project Settings → Database → Connection Pooler.
    - RUNTIME string (app runtime, port 6543, Transaction mode):
        postgresql://postgres.{ref}:{password}@aws-{region}.pooler.supabase.com:6543/postgres?sslmode=require
    - SESSION string (for schema/seed DDL only, port 5432, Session mode):
        postgresql://postgres.{ref}:{password}@aws-{region}.pooler.supabase.com:5432/postgres?sslmode=require
    Note: both use the SAME pooler hostname (aws-{region}.pooler.supabase.com) — only the port differs. The direct host db.{ref}.supabase.co is IPv6-only and will NOT work from Render (IPv4). Never use the direct host.

    STEP 3: Verify pool connectivity from laptop.
    Set DATABASE_URL in your local .env to the RUNTIME string (6543):
        DATABASE_URL="postgresql://postgres.{ref}:{password}@aws-{region}.pooler.supabase.com:6543/postgres?sslmode=require"
    Then run:
        uv run python -c "
        from app.db.supabase import get_connection
        with get_connection() as conn:
            row = conn.execute('SELECT 1 AS ok').fetchone()
            print('Pool OK:', row)
        "
    Expected output: Pool OK: (1,)
    This confirms: (a) prepare_threshold=None already set on the pool works with the Supavisor, (b) IPv4 connectivity from laptop to pooler is good.

    STEP 4: Apply schema and seed using the SESSION string (5432). Per D-15, DDL over the session pooler (5432) is preferred. Run BOTH commands with the session URL:
        DATABASE_URL="postgresql://postgres.{ref}:{password}@aws-{region}.pooler.supabase.com:5432/postgres?sslmode=require" \
          uv run python -m app.db.bootstrap --reset
        DATABASE_URL="postgresql://postgres.{ref}:{password}@aws-{region}.pooler.supabase.com:5432/postgres?sslmode=require" \
          uv run python -m app.db.seed
    If bootstrap also works on 6543, note it (this resolves ⚠ A4 — see CONFIRM below). D-15 says prefer 5432 for DDL; keep using 5432 for future schema changes regardless.

    STEP 5: Restore .env to RUNTIME string (6543) and run integration tests:
        uv run pytest -m integration -x -q
    These need a live DB — they will now run (not skip). Note any failures.

    CONFIRM (required in resume-signal):
