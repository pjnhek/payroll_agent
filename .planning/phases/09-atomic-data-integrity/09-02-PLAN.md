<objective>
Wire the three DATA-01 atomicity boundaries locked in 09-CONTEXT.md (D-9-04 through
D-9-08) into `app/pipeline/orchestrator.py`, using the `repo.get_connection()` +
`conn.transaction()` seam that 09-01 made mockable offline. This is pure wiring —
no new business logic, no changed call order — around code that already does the
right work in the right sequence: `_run_stages`' process branch, `_clarify`'s three
AWAITING_REPLY exit paths, `_defer_field_regression_clarification`'s
`set_clarified_fields` write (the resume-path field-regression re-clarification
helper called from BOTH `resume_pipeline` Round-1 and Round-2 branches — D-9-06's
canonical_refs cross-reference to `07.5-CONTEXT.md`'s write sequences), and
`_deliver`'s finalize sequence each become ONE atomic unit, with the run-status
write always LAST inside it where a status write is present (D-9-02). The
provider/LLM calls inside `_clarify`/`_deliver`/`_defer_field_regression_clarification`
stay explicitly OUTSIDE any transaction (D-9-01) — `gateway.send_outbound`'s own
D-13c reserved-before-send/flip-after-send lifecycle is UNCHANGED; only the writes
that come AFTER an external side effect returns (or, for
`_defer_field_regression_clarification`, strictly BEFORE the `_clarify` call it
makes) get a transaction wrapped around them.

Purpose: close audit finding HIGH-03 — a crash between any two of these writes today
leaves a half-written run (e.g. line items replaced but status still `extracting`,
or a confirmation genuinely sent but the run stuck at APPROVED forever), INCLUDING
the resume-path field-regression re-clarification write (`set_clarified_fields`)
that today auto-commits independently before `_clarify`'s two LLM calls + send — a
crash there leaves `clarified_fields` durably marked "asked" with no clarification
ever sent and the run's status un-advanced, on the same MONEY-03 path that had three
rounds of review failures in Phase 7.5. After this plan, every such crash leaves the
run wholly un-advanced, provable by fault injection against a real/local Postgres
connection.

Output: `_run_stages`, `_clarify`, `_defer_field_regression_clarification`, and
`_deliver` in `app/pipeline/orchestrator.py` each wrap their existing write
sequences in one `conn.transaction()` block; `_write_aliases_if_safe` gains a
`conn=` parameter; `tests/test_atomic_persist.py` proves SC1 via real-connection
fault injection.
</objective>
