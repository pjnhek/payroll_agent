<objective>
Close the repository boundaries that can route stale clarification replies or expose mutable/overbroad email state.

Purpose: exactly-once sending depends on the current reply epoch being the sole live clarification context, and operator review depends on a safe projection whose callers cannot accidentally receive payroll/client body content.
Output: epoch-scoped header routing, body-free review facts, a fail-closed legacy state seam, and regression coverage for each boundary.
</objective>

## Artifacts this phase produces

- A current-epoch predicate on the awaiting-reply header query, with late-reply observability intentionally kept separate.
- A bounded delivery-review SQL projection that omits body text while retaining purpose, identity, timing, category/attempt facts, and attachment metadata.
- A constrained outbound-only state transition helper or an inert retired compatibility mutator, with inbound/arbitrary-state regressions.
