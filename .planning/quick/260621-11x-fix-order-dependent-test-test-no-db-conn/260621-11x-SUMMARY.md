## What Was Done

### Task 1 â€” AST-based import guard (tests/test_status_drift.py)

**Root cause:** `test_no_db_connection_needed` asserted `"app.db.supabase" not in sys.modules`. `sys.modules` is process-global: when `test_seed_roundtrip.py` runs first in the same pytest process (which happens when `DATABASE_URL` is set), `app.db.supabase` is already imported and in `sys.modules`. The test was failing even though `test_status_drift.py` itself never imports the DB layer.

**Fix:** Replaced the `sys.modules` check with AST inspection:
- Added `import ast` to module-level imports.
- `test_no_db_connection_needed` now reads its own source via `pathlib.Path(__file__).read_text()`, parses it with `ast.parse`, walks all `Import` and `ImportFrom` nodes, and asserts none reference `app.db.supabase`, `psycopg`, or `psycopg_pool`.
- Removed the local `import sys` inside the function body (no longer needed).
- Updated docstring to explain the new mechanism and why `sys.modules` is not used.

### Task 2 â€” close_pool() + seed CLI try/finally (app/db/supabase.py, app/db/seed.py)

**supabase.py changes:**
- Added `close_pool()` â€” checks if `_pool is not None`, calls `_pool.close()`, then sets `_pool = None`. Idempotent; safe to call multiple times or when pool was never opened.
- Added `open=True` as explicit keyword argument to `ConnectionPool(...)` to silence the psycopg_pool DeprecationWarning about default behavior changing in a future release (zero behavioral change â€” pool already opened immediately).
- Updated module docstring Public API block to list `close_pool()`.

**seed.py changes:**
- Wrapped `__main__` block in `try/finally`. `close_pool()` is called in the `finally` clause whether `seed()` succeeds or raises. The import is deferred inside the `finally` block consistent with seed.py's existing pattern (dry-run imports never pull in the DB layer).

**app/db/bootstrap.py â€” unchanged:** Uses `psycopg.connect(...)` as a direct context manager (not the pool), closes on `with` block exit. No `close_pool()` call needed.
