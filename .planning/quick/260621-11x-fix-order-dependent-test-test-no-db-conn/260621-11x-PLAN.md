<objective>
Fix one order-dependent test failure and add clean pool shutdown to the seed CLI.

Purpose: The test `test_no_db_connection_needed` asserts on `sys.modules`, which is
process-global. When seed-roundtrip integration tests precede it in the same pytest
process (which happens when DATABASE_URL is set), `app.db.supabase` is already in
`sys.modules`, making the assertion false even though the drift test file itself never
imports the DB layer. Separately, `python -m app.db.seed` leaks the ConnectionPool
singleton at exit, causing a 5-second thread-shutdown warning.

Output: A corrected test using AST inspection; a `close_pool()` function in
`app/db/supabase.py`; and a `finally` block in `seed.py`'s CLI entrypoint.
</objective>

**app/db/supabase.py — add close_pool():**

After the `get_connection()` context manager, add a new public function `close_pool()`.
Update the module docstring's "Public API" comment block to include the new function.

`close_pool()` must:
1. Reference the module-level `_pool` global.
2. If `_pool` is not None, call `_pool.close()` to drain active connections and
   stop the background worker thread.
3. Set `_pool = None` so a subsequent `get_pool()` call can recreate a fresh pool
   (important for tests that mock the pool).
4. If `_pool` is already None, return silently (idempotent).

Also, while touching supabase.py: add `open=True` as an explicit keyword argument
to the `ConnectionPool(...)` call in `get_pool()`. This silences the psycopg_pool
DeprecationWarning about the `open` parameter defaulting behavior changing in a
future release. It is a one-word addition with zero behavioral change (the pool
already opens immediately; this just makes it explicit).

**app/db/seed.py — wrap CLI entrypoint in try/finally:**

The `if __name__ == "__main__":` block currently reads:

    if __name__ == "__main__":
        seed()

Rewrite it to:

    if __name__ == "__main__":
        try:
            seed()
        finally:
            from app.db.supabase import close_pool
            close_pool()

The `finally` block runs `close_pool()` whether `seed()` succeeds or raises an
exception. The import is deferred inside the block intentionally — consistent with
how `seed.py` already defers `from app.db.supabase import get_connection` to the
live path inside `seed()` — so a dry-run import of seed.py never pulls in the DB
layer. The `close_pool()` call is safe even on the dry-run path because it checks
`_pool is None` (idempotent).

**app/db/bootstrap.py — NO CHANGES NEEDED:**

bootstrap.py uses `psycopg.connect(...)` as a direct context manager (line 88),
which closes the connection on exit. It does not call `get_pool()` or
`get_connection()` at any point. Adding `close_pool()` there would be a spurious
call against a None singleton — skip it.
